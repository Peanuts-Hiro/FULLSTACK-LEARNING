## 学習した技術要素

### 1. HTML5

#### セマンティックHTML
- `<header>`, `<nav>`, `<main>`, `<section>`, `<footer>`の使い分け
- セマンティックタグを使うことでHTMLの構造が分かりやすくなる
- SEO（検索エンジン最適化）にも有利

#### 基本的なタグ
- 見出しタグ（`<h1>` - `<h3>`）の階層構造
- リスト（`<ul>`, `<li>`）
- リンク（`<a>`）のhref属性とtarget属性
- コンテナ（`<div>`, `<span>`）の使い分け

#### ベストプラクティス
- `<!DOCTYPE html>`宣言の重要性
- `<meta charset="UTF-8">`で文字化け防止
- `<meta name="viewport">`でレスポンシブ対応の準備
- `rel="noopener noreferrer"`でセキュリティ対策

---

### 2. CSS3

#### CSS変数（カスタムプロパティ）
```css
:root {
    --primary-color: #2c3e50;
    --spacing-md: 24px;
}
```
- 色やサイズを一元管理できる
- 変更が容易で保守性が高い
- デザインシステムの基礎

#### リセットCSS
```css
* {
    margin: 0;
    padding: 0;
    box-sizing: border-box;
}
```
- ブラウザのデフォルトスタイルをリセット
- `box-sizing: border-box`でサイズ計算を簡単に

#### Flexbox
5つの異なるFlexboxパターンを実装:
1. **ナビゲーション**: `justify-content: space-between`
2. **ヒーロー**: `flex-direction: column` + `justify-content: center`
3. **スキルカード**: `flex-wrap: wrap` + `gap`
4. **プロジェクトカード**: スキルカードと同様
5. **コンタクトリンク**: `justify-content: center`

#### ホバーエフェクト
4つの異なるホバーパターンを実装:
1. **ナビゲーションリンク**: 下線表示（`border-bottom`）
2. **スキルカード**: 拡大 + 背景色変更 + 影強調
3. **プロジェクトカード**: 浮き上がり（`translateY`）
4. **コンタクトアイコン**: 回転（`rotate`）

#### アニメーション
- `transition: all 0.3s ease`で滑らかな変化
- `transform`（scale, translateY, rotate）で変形
- `box-shadow`で奥行き感を演出

## 技術的な気づき

### うまくいったこと ✅

1. **CSS変数の活用**
   - 色やサイズを一元管理できて変更が容易
   - コードの可読性が向上

2. **Flexboxによるレイアウト**
   - メディアクエリなしでレスポンシブ対応
   - `flex-wrap`と`flex-basis`の組み合わせが便利

3. **gapプロパティ**
   - marginより簡潔に余白を設定できる
   - 端の処理が不要

4. **transformによるアニメーション**
   - GPU処理で高速
   - レイアウトに影響しない

5. **学習用コメントの充実**
   - 後から見返したときに理解しやすい
   - 初学者にも優しいコード

---

### 改善できる点 🔧

1. **レスポンシブ対応の強化**
   - 現在は`flex-wrap`のみで対応
   - メディアクエリを追加してより細かい調整が必要

2. **アクセシビリティの向上**
   - キーボード操作への対応
   - スクリーンリーダーへの配慮（aria属性）

3. **パフォーマンス**
   - 現在は画像を絵文字で代替しているが、実際の画像を使う場合は最適化が必要
   - 画像の遅延読み込み（lazy loading）

4. **インタラクティブ機能**
   - 現在は静的なサイト
   - JavaScriptでダイナミックな機能を追加したい

5. **ブラウザ互換性**
   - モダンブラウザのみを対象としている
   - 古いブラウザへの対応（プレフィックスなど）

## 学習の振り返り

### 最も重要だった学び

**「設計の重要性」**

実装前に要件定義と設計を行うことで:
- 実装がスムーズに進んだ
- 迷いが少なく、コードの品質が向上
- 全体像を把握できた

最初は面倒に感じたが、結果的に時間短縮につながった。

---

### 難しかった点

1. **Flexboxの主軸と交差軸の理解**
   - `justify-content`と`align-items`の使い分けに最初は混乱
   - `flex-direction`で主軸が変わることを理解するのに時間がかかった

2. **transitionの設定場所**
   - 最初はホバー状態に書いてしまった
   - 通常状態に書く必要があることを学んだ

3. **transformの複数指定**
   - 別々の行に書くと上書きされることを知らなかった
   - 1行にまとめる必要がある

---

### 楽しかった点

1. **ホバーエフェクトの実装**
   - 要素が動くのが楽しい
   - 値を変えて実験するのが面白かった

2. **Flexboxでのレイアウト**
   - 従来のfloatより直感的
   - 中央揃えが簡単にできて感動

3. **CSS変数の便利さ**
   - 色を一括変更できるのが便利
   - デザインシステムの考え方を学べた
