# フェーズ1: HTML構造の実装 - 思考と学習ポイント

## 実装時の思考プロセス

### 1. 「セマンティック」を最優先に考えた

**思考**: 初学者は`<div>`を多用しがち。しかし、HTMLは「意味」で書くべき。

#### なぜセマンティックタグを使うのか？

**悪い例（div地獄）**:
```html
<div class="header">
  <div class="nav">
    <div class="logo">サイト名</div>
    <div class="menu">
      <div class="menu-item">About</div>
      <div class="menu-item">Skills</div>
    </div>
  </div>
</div>
```

**問題点**:
- どれが何なのか分からない
- SEOに悪影響（検索エンジンが構造を理解できない）
- アクセシビリティが低い（スクリーンリーダーが読みにくい）
- 保守性が低い（他の人が見ても分からない）

**良い例（セマンティックHTML）**:
```html
<header>
  <nav>
    <h1>サイト名</h1>
    <ul>
      <li><a href="#about">About</a></li>
      <li><a href="#skills">Skills</a></li>
    </ul>
  </nav>
</header>
```

**メリット**:
- 一目で構造が分かる
- SEOに有利（検索エンジンが理解しやすい）
- アクセシビリティが高い
- 保守性が高い

### 2. コメントを「教科書」として書いた

**思考**: 初学者がコードを見返したとき、「なぜこう書いたか」が分かるように。

#### コメントの書き方の工夫

**悪いコメント**:
```html
<!-- ヘッダー -->
<header>
  <!-- ナビ -->
  <nav>
    <!-- タイトル -->
    <h1>Your Name</h1>
  </nav>
</header>
```

**問題点**: 「何をしているか」しか書いていない（コードを見れば分かる）

**良いコメント**:
```html
<!-- ===== ヘッダーセクション ===== -->
<!-- headerタグ: ページのヘッダー部分を表すセマンティックタグ -->
<header>
    <!-- navタグ: ナビゲーションリンクをまとめるセマンティックタグ -->
    <nav>
        <!-- h1タグ: ページの最も重要な見出し（1ページに1つ） -->
        <h1>Your Name</h1>
    </nav>
</header>
```

**良い点**:
- タグの**意味**を説明
- **なぜそのタグを使うのか**を説明
- 学習資料として機能する

**コメントの3つのレベル**:

1. **セクションコメント**: `<!-- ===== ヘッダーセクション ===== -->`
   - 大きなまとまりを示す
   - 視覚的に分かりやすい

2. **説明コメント**: `<!-- headerタグ: ページのヘッダー部分を表すセマンティックタグ -->`
   - タグの意味を説明
   - 学習用

3. **技術コメント**: `<!-- target="_blank": 新しいタブで開く -->`
   - 属性の意味を説明
   - 実践的な知識

### 3. 段階的な構造を意識した

**思考**: HTMLは「大きなブロック」から「小さな要素」へ。

#### HTML構造の階層

```
<!DOCTYPE html>              ← ドキュメントタイプ宣言
└── <html>                   ← ルート要素
    ├── <head>               ← メタ情報
    │   ├── <meta>           ← 文字コード、ビューポート
    │   ├── <title>          ← ページタイトル
    │   └── <link>           ← CSSファイル
    └── <body>               ← 表示される内容
        ├── <header>         ← ヘッダー
        │   └── <nav>        ← ナビゲーション
        ├── <main>           ← メインコンテンツ
        │   ├── <section id="hero">    ← ヒーロー
        │   ├── <section id="about">   ← About
        │   ├── <section id="skills">  ← Skills
        │   ├── <section id="projects"> ← Projects
        │   └── <section id="contact">  ← Contact
        └── <footer>         ← フッター
```

**設計原則**:
1. **外側から内側へ**: まず全体構造、次に各セクション、最後に詳細
2. **意味のまとまりで分ける**: セクションごとに機能が明確
3. **一貫性を保つ**: すべてのセクションで同じパターン

---

### 4. id属性の戦略的な使用

**思考**: id属性は2つの重要な役割がある。

#### id属性の役割

**役割1: ページ内リンクのターゲット**

```html
<!-- ナビゲーション -->
<nav>
  <ul>
    <li><a href="#about">About</a></li>
    <li><a href="#skills">Skills</a></li>
  </ul>
</nav>

<!-- リンク先のセクション -->
<section id="about">
  <h2>About Me</h2>
</section>

<section id="skills">
  <h2>Skills</h2>
</section>
```

**仕組み**:
- `href="#about"`: `#`はページ内リンクを意味する
- `id="about"`: このセクションがリンク先
- クリックすると、そのセクションにスムーズにスクロール

**役割2: CSSセレクタとして使用**

```css
/* id属性を使ったCSS */
#hero {
  background: blue;
}

#about {
  padding: 60px;
}
```

**idとclassの使い分け**:
- **id**: ページ内で1つだけ（ユニーク）
- **class**: 複数の要素に使える

### 5. リンクのセキュリティを考慮した

**思考**: 外部リンクにはセキュリティ対策が必要。

#### target="_blank"の危険性

**危険なコード**:
```html
<a href="https://example.com" target="_blank">
  外部サイト
</a>
```

**問題点**:
- 新しいタブで開いたページから、元のページを操作できる（タブナビング攻撃）
- `window.opener`を使って元のページにアクセス可能

**安全なコード**:
```html
<a href="https://example.com"
   target="_blank"
   rel="noopener noreferrer">
  外部サイト
</a>
```

**解説**:
- `target="_blank"`: 新しいタブで開く
- `rel="noopener"`: 新しいタブから元のページにアクセスさせない
- `rel="noreferrer"`: リファラー情報を送信しない（プライバシー保護）

**覚え方**: 「外部リンクには必ずセット」

## 学習者がくみ取るべき内容

### 1. セマンティックHTMLの実践

#### 主要なセマンティックタグ

| タグ | 意味 | 使用箇所 |
|------|------|----------|
| `<header>` | ページのヘッダー | サイトの上部 |
| `<nav>` | ナビゲーション | メニュー |
| `<main>` | メインコンテンツ | ページの主要部分 |
| `<section>` | セクション（まとまり） | About, Skills等 |
| `<article>` | 独立したコンテンツ | ブログ記事等 |
| `<aside>` | 補足情報 | サイドバー |
| `<footer>` | フッター | ページの下部 |

#### なぜこのタグを選ぶのか？

**例1: ヘッダー**
```html
<!-- なぜheaderを使うのか？ -->
<header>  <!-- ← ページの「頭」部分だから -->
  <nav>   <!-- ← 「ナビゲーション」だから -->
    <h1>サイト名</h1>  <!-- ← 最も重要な「見出し」だから -->
  </nav>
</header>
```

**例2: メインコンテンツ**
```html
<!-- なぜmainを使うのか？ -->
<main>  <!-- ← ページの「主要」部分だから -->
  <section id="about">  <!-- ← 「セクション」（まとまり）だから -->
    <h2>About Me</h2>   <!-- ← セクションの「見出し」だから -->
    <p>説明文</p>        <!-- ← 「段落」だから -->
  </section>
</main>
```

**判断基準**:
1. この要素の**意味**は何か？
2. その意味に**最も近いタグ**は何か？
3. 分からなければ`<div>`（最後の手段）

### 2. HTML文書の基本構造

#### 必須の要素

```html
<!DOCTYPE html>  <!-- ← HTML5であることを宣言 -->
<html lang="ja"> <!-- ← 日本語のページであることを宣言 -->
<head>
    <!-- 文字エンコーディング -->
    <meta charset="UTF-8">

    <!-- レスポンシブデザインの準備 -->
    <meta name="viewport" content="width=device-width, initial-scale=1.0">

    <!-- ページタイトル（タブに表示） -->
    <title>あなたの名前 - ポートフォリオ</title>

    <!-- CSSファイルの読み込み -->
    <link rel="stylesheet" href="style.css">
</head>
<body>
    <!-- ここに表示される内容 -->
</body>
</html>
```

#### 各要素の意味

**`<!DOCTYPE html>`**:
- 「このファイルはHTML5です」という宣言
- 必ず1行目に書く
- 大文字小文字は問わない

**`<html lang="ja">`**:
- `lang="ja"`: 日本語のページであることを宣言
- 検索エンジンとブラウザが言語を認識
- スクリーンリーダーが正しい言語で読み上げ

**`<meta charset="UTF-8">`**:
- 文字エンコーディングをUTF-8に設定
- 日本語を正しく表示するために必須
- **必ずhead内の最初に書く**

**`<meta name="viewport" ...>`**:
- スマホ対応の準備
- `width=device-width`: デバイスの幅に合わせる
- `initial-scale=1.0`: 初期ズームを100%に

**`<link rel="stylesheet" href="style.css">`**:
- 外部CSSファイルを読み込む
- `rel="stylesheet"`: スタイルシートであることを示す
- `href="style.css"`: ファイルのパス

### 3. ページ内リンクの仕組み

#### 基本の構造

```html
<!-- ステップ1: ナビゲーションにリンクを作る -->
<nav>
  <ul>
    <li><a href="#about">About</a></li>
    <li><a href="#skills">Skills</a></li>
    <li><a href="#projects">Projects</a></li>
  </ul>
</nav>

<!-- ステップ2: リンク先にid属性を付ける -->
<section id="about">
  <h2>About Me</h2>
</section>

<section id="skills">
  <h2>Skills</h2>
</section>

<section id="projects">
  <h2>Projects</h2>
</section>
```

#### 動作の流れ

1. ユーザーが「About」リンクをクリック
2. `href="#about"`により、`id="about"`の要素を探す
3. その要素までスクロール

#### スムーススクロールの追加（CSS）

```css
html {
  scroll-behavior: smooth;
}
```

**効果**: カクカクではなく、なめらかにスクロール

### 4. リストとリンクの正しい使い方

#### ナビゲーションの標準的な構造

```html
<nav>
  <ul>                          <!-- ul: 順序なしリスト -->
    <li>                        <!-- li: リスト項目 -->
      <a href="#about">About</a>  <!-- a: リンク -->
    </li>
    <li>
      <a href="#skills">Skills</a>
    </li>
  </ul>
</nav>
```

#### なぜこの構造なのか？

**理由1: セマンティック**
- ナビゲーションは「項目のリスト」
- リストなので`<ul>`と`<li>`を使う

**理由2: アクセシビリティ**
- スクリーンリーダーが「リスト」として認識
- 「3つの項目があります」と読み上げる

**理由3: スタイリングしやすい**
- CSS で`li { display: inline-block; }`で横並びに
- または Flexbox で柔軟にレイアウト

### 5. コンテンツの構造化

#### セクションの基本パターン

```html
<section id="セクション名">
  <h2>セクションの見出し</h2>

  <div class="コンテナ名">
    <!-- ここに具体的な内容 -->
  </div>
</section>
```

#### 実例: Skillsセクション

```html
<section id="skills">
  <!-- セクションの見出し -->
  <h2>Skills</h2>

  <!-- スキルカードのコンテナ -->
  <div class="skills-container">

    <!-- 各スキルカード -->
    <div class="skill-card">
      <div class="skill-icon">🌐</div>
      <h3>HTML/CSS</h3>
      <p>セマンティックHTML5とモダンCSSの基礎を学習中です。</p>
    </div>

    <div class="skill-card">
      <div class="skill-icon">⚡</div>
      <h3>JavaScript</h3>
      <p>基本的な構文とDOM操作を学んでいます。</p>
    </div>

  </div>
</section>
```

#### 構造の意図

**3層構造**:
1. `<section>`: 大きなまとまり
2. `.skills-container`: カードを入れる箱（Flexboxで配置）
3. `.skill-card`: 個々のカード（再利用可能なコンポーネント）

**なぜdivを使うのか？**:
- `.skills-container`: 「スキルカードの容器」という意味的な役割がない → `<div>`
- `.skill-card`: 「カード」という視覚的な要素 → `<div>`

**セマンティックタグとdivの使い分け**:
- **セマンティックタグ**: 意味がある（header, nav, section等）
- **div**: 意味がない、見た目のグループ化のみ

## よくある間違いと解決策

### 間違い1: divだらけのHTML

**悪い例**:
```html
<div class="header">
  <div class="title">サイト名</div>
</div>
```

**修正**:
```html
<header>
  <h1>サイト名</h1>
</header>
```

**理由**: 意味があるなら、セマンティックタグを使う

---

### 間違い2: h1が複数ある

**悪い例**:
```html
<h1>サイト名</h1>
<section>
  <h1>About</h1>  <!-- ダメ！ -->
</section>
```

**修正**:
```html
<h1>サイト名</h1>
<section>
  <h2>About</h2>  <!-- セクションの見出しはh2 -->
</section>
```

**理由**: h1はページに1つだけ（最も重要な見出し）

---

### 間違い3: id属性の重複

**悪い例**:
```html
<section id="section">...</section>
<section id="section">...</section>  <!-- ダメ！ -->
```

**修正**:
```html
<section id="about">...</section>
<section id="skills">...</section>
```

**理由**: idはページ内で一意（ユニーク）でなければならない

---

### 間違い4: 閉じタグの忘れ

**悪い例**:
```html
<section>
  <h2>About
  <p>説明文</p>
</section>
```

**修正**:
```html
<section>
  <h2>About</h2>  <!-- 閉じタグを忘れずに -->
  <p>説明文</p>
</section>
```

**予防策**: エディタの自動補完機能を使う
