# ToDoリストアプリ 設計書

## 1. 全体構成

### 1.1 ファイル構成
```
week01-todo-app/
├── index.html    # HTML構造（画面の骨組み）
├── style.css     # CSS（見た目・デザイン）
└── script.js     # JavaScript（動作・機能）
```

### 1.2 技術スタック
- **HTML5**: ページ構造
- **CSS3**: スタイリング
- **Vanilla JavaScript**: 機能実装（ライブラリなし）

---

## 2. HTML設計（index.html）

### 2.1 画面構成
```
┌─────────────────────────┐
│  ToDoリスト           │ ← タイトル
├─────────────────────────┤
│ [入力欄      ] [追加] │ ← 入力エリア
├─────────────────────────┤
│ ☐ タスク1    [削除]   │ ← タスクリスト
│ ☐ タスク2    [削除]   │
│ ☐ タスク3    [削除]   │
└─────────────────────────┘
```

### 2.2 必要なHTML要素
```html
- <h1>: タイトル
- <input type="text">: タスク入力欄
- <button>: 追加ボタン
- <ul>: タスクリストのコンテナ
- <li>: 各タスク項目（動的に生成）
- <button>: 削除ボタン（各タスクごと）
```

### 2.3 ID/Class設計
| 要素 | ID/Class | 用途 |
|------|----------|------|
| 入力欄 | `#taskInput` | JavaScriptで値を取得 |
| 追加ボタン | `#addButton` | クリックイベント設定 |
| タスクリスト | `#taskList` | タスクを追加する親要素 |
| 削除ボタン | `.deleteBtn` | 各タスクの削除ボタン |

## 3. CSS設計（style.css）

### 3.1 デザインコンセプト
- **シンプル**: 初心者が理解しやすいレイアウト
- **見やすさ**: 適切な余白と文字サイズ
- **使いやすさ**: ボタンが押しやすいサイズ

### 3.2 レイアウト
```
- 中央寄せ: コンテンツを画面中央に配置
- 最大幅: 600px（大画面でも広がりすぎない）
- 余白: 適切なpadding/margin
```

### 3.3 スタイル方針
```css
- フォント: sans-serif（読みやすい）
- 配色: シンプルな白背景 + アクセントカラー
- ボタン: hover時に色が変わる（視覚的フィードバック）
- リスト: 各項目を区切る線（境界線）
```

## 4. JavaScript設計（script.js）

### 4.1 データ構造
タスクは配列で管理せず、**DOMに直接追加**する方式
```javascript
// タスクはDOM（HTML要素）として存在
// <li>タスク名 <button>削除</button></li>
```

### 4.2 機能設計

#### 機能1: タスク追加
```
【トリガー】追加ボタンをクリック
【処理フロー】
1. 入力欄の値を取得
2. 空チェック（空なら何もしない）
3. <li>要素を新規作成
4. タスク名と削除ボタンを<li>に追加
5. <ul>に<li>を追加
6. 入力欄をクリア
```

#### 機能2: タスク表示
```
【方法】
- HTMLに<ul id="taskList"></ul>を用意
- JavaScriptで動的に<li>を追加
```

#### 機能3: タスク削除
```
【トリガー】削除ボタンをクリック
【処理フロー】
1. クリックされた削除ボタンを特定
2. その親要素（<li>）を取得
3. <li>をDOMから削除
```

### 4.3 イベント設計
| イベント | 対象 | 処理内容 |
|----------|------|----------|
| click | 追加ボタン | タスク追加処理を実行 |
| click | 削除ボタン | そのタスクを削除 |

### 4.4 関数設計
```javascript
// 関数1: タスク追加
function addTask() {
  // 入力値取得 → 空チェック → DOM操作
}

// 関数2: タスク削除
function deleteTask(event) {
  // クリックされた要素の親を削除
}
```

## 5. 処理フロー図

### タスク追加の流れ
```
ユーザーがテキスト入力
       ↓
「追加」ボタンをクリック
       ↓
入力値が空？ → YES → 何もしない
       ↓ NO
新しい<li>を作成
       ↓
タスク名と削除ボタンを追加
       ↓
<ul>に<li>を挿入
       ↓
入力欄をクリア
       ↓
画面に新しいタスクが表示される
```

### タスク削除の流れ
```
ユーザーが削除ボタンをクリック
       ↓
どの削除ボタンか特定
       ↓
その親の<li>を取得
       ↓
<li>をDOMから削除
       ↓
画面からタスクが消える
```

## 6. 実装時の注意点

### 6.1 入力検証
- 空文字チェック: `if (input.value.trim() === "")`を使用
- `trim()`で前後の空白を除去

### 6.2 DOM操作
- `document.createElement()`: 要素作成
- `appendChild()`: 要素追加
- `remove()`: 要素削除

### 6.3 イベント処理
- 追加ボタン: `addEventListener('click', addTask)`
- 削除ボタン: 動的に生成するため、各ボタンにイベント設定
